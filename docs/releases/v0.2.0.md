# Release v0.2.0

**Date:** 2026-07  
**Previous release:** [v0.1.0](./v0.1.0.md)

## Summary vs previous release

Focus on **operability and clearer auth feedback**: configuration via environment variables (no hard-coded production URLs), documentation for local development and Render migration, and login/register errors that stay **inside the form** with friendlier wording—instead of raw or full-page errors.

## Features (new vs 0.1.0)

- `REACT_APP_API_URL` and backend `CORS_ALLOWED_ORIGINS` driven by env
- Frontend config module (`frontend/src/config.js`)
- Axios interceptors that normalize API errors for the UI
- Compact auth error component under password fields
- Plain-language messages for duplicate email/username and bad credentials
- Backend duplicate checks and friendlier register/login messages

## Bug fixes (vs 0.1.0)

- Auth errors no longer dump users onto a separate HTML-style error experience
- Registration/login feedback visible on the same screen

## Documentation (new vs 0.1.0)

- [docs/development.md](../development.md) — local runbook
- [docs/deployment/](../deployment/) — architecture + **Render migration checklist**
- README links to local setup guide

## Deployment / infrastructure (vs 0.1.0)

- Prepared migration path **Railway → Render** (checklist, env var lists)
- Aiven MySQL connection pattern documented (SSL / CA optional)
- GitHub Actions called out as optional; native Render/Vercel deploys preferred

## Upgrade notes

- Set `REACT_APP_API_URL` on Vercel for production builds
- Set `CORS_ALLOWED_ORIGINS` on the API host to the exact Vercel origin
- Copy `backend/.env.example` / `frontend/.env.example` for local work
