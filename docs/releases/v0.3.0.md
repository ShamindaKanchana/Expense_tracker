# Release v0.3.0

**Date:** 2026-07  
**Previous release:** [v0.2.0](./v0.2.0.md)  
**Status:** **Current**

## Summary vs previous release

This release turns the product into a clearer **username-first** free-tier app: people can sign up without email, recover orientation with **Info/Help** and account tools, and survive common free-hosting and CI pain. Also ships full **API** and **database** docs, release logging, and production build reliability on Vercel.

| Area | v0.2.0 | v0.3.0 |
|------|--------|--------|
| Signup | Username + **email** + password | **Username + password** (email optional / unused in UI) |
| Login field | Email | **Username or email** |
| After login identity | Not shown in UI | **Account** page shows username; change password |
| Session expired | Dashboard errors / empty sample data | Redirect to **sign-in** with message |
| Auth education | Minimal | **Info · Help** on login/register; username field tip on signup |
| Docs | Dev + deployment | + **API**, **database/ER**, **releases** |
| Vercel build | Could fail (ESLint / TS / jest env) | Pinned TypeScript; ESLint fixes |
| Monthly report years | Dropdown lost current year after selecting past year | Dropdown always includes current year |

## Features (new vs 0.2.0)

### Authentication & accounts

- Username + password registration (no email required)
- Login accepts **username or email** (legacy email users still work)
- `users.email` nullable; startup `ALTER` for existing DBs
- **Account** page (`/account`): view username (and email if present), change password, sign out
- Navbar **Account** chip (avatar initial + username on desktop)
- `GET /api/auth/me` fixed; `POST /api/auth/change-password` added
- Compact **Info** and **Help** panels on sign-in / sign-up (app purpose, how to register, free-tier wake delays)
- Signup username field: small **(i)** tip — remember username for next sign-in

### UX / product polish

- Global API **401** handling → clear session → login (except login/register wrong credentials)
- Session-expired banner message on the login form

## Bug fixes (vs 0.2.0)

- Vercel **Deployment Failed** from ESLint unused vars under `CI=true`
- Vercel/ESLint failure: `jest/globals is unknown` (TypeScript 7 / jest preset) — pin TypeScript 4.9.5, drop `react-app/jest` from production ESLint config
- Monthly Report year dropdown no longer hides the **current year** after selecting an older year
- Expired JWT no longer leaves Dashboard on “Token has expired. Using sample data.”

## Documentation (new vs 0.2.0)

- [docs/api.md](../api.md) — endpoints, public vs private, request/response shapes
- [docs/database.md](../database.md) — ER diagram, tables, models
- [docs/releases/](./README.md) — this release log
- API docs updated for username login and change-password

## Deployment / infrastructure (vs 0.2.0)

- Backend live path toward **Render** free tier (`onrender.com`)
- Frontend remains **Vercel**; must rebuild when `REACT_APP_API_URL` changes
- Free-tier wake delays documented in Help (first request after idle can take 30–60s)

## Upgrade notes

### Operators (Render / Vercel / DB)

1. Deploy **backend** first so `ALTER` on `users.email` and new auth routes apply.
2. Deploy **frontend** so Account, Help, and username signup UI ship.
3. Confirm env:
   - Vercel: `REACT_APP_API_URL=https://<api-host>/api`
   - Render: `CORS_ALLOWED_ORIGINS=https://<vercel-app>`
4. Existing users: still sign in with **email or username**; new users use **username only**.

### End users

- New accounts: remember your **username** (no email reset).
- Change password anytime under **Account**.
- If login is slow once after a long idle, wait and retry (free hosting sleep).
