## Summary vs previous release

This release makes Expense Tracker **trilingual** (English, Sinhala, Tamil) across the main user app, and ships **admin-approved password reset** plus related auth UX polish that landed after v0.3.0.

| Area | v0.3.0 | v0.4.0 |
|------|--------|--------|
| UI language | English only | **EN · සිං · த** with switcher + `localStorage` |
| App brand | Expense Tracker | EN: Expense Tracker · SI: **වියදම් පොත** · TA: **செலவு கணக்கு** |
| Forgot password | Not present | Admin-approved request + one-time code reset |
| Month labels | English (or broken SI/TA truncation) | Natural full + short month names per locale |

## Features

### Internationalization (i18n)

- `react-i18next` / `i18next` with locale files: `en.json`, `si.json`, `ta.json`
- **Language switcher** (EN \| සිං \| த) on auth screens, navbar, and Account
- Preference remembered in `localStorage` (`expense_tracker_lang`)
- Translated: login/register, forgot/reset password, navbar, dashboard, add expense, monthly report, account, Info/Help, common errors
- **Categories** stay as English keys in the DB (`Food`, `Transport`, …); **labels** translate in the UI
- **Month names:** full + natural short forms (no more first-3-letter slicing of Sinhala/Tamil)

### Password reset (admin-approved)

- User: `/forgot-password` → pending request → post-submit next steps (no re-submit spam)
- Admin: list / approve / reject; one-time code shown once on approve
- User: `/reset-password` with username + code + new password
- Related UX: field/button spacing, request-submitted state, username prefilled on reset

- Admin still needs `ADMIN_USERNAME` / `ADMIN_PASSWORD` if using admin panel
